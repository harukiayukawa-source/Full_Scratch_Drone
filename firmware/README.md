# firmwareフォルダの中身について
## drone_PID_1フォルダ
VSCode内の拡張機能「PlatformIO」を利用してコーディング・ビルド・コンパイルを行いました。<br>
includeフォルダ, libフォルダ, platformio.iniファイル, srcフォルダ

## includeフォルダ
特に使っていません。

## libフォルダ
特に使っていません。

## platformio.iniファイル
書き込み先のボード種やシリアル通信のbps、インクルードするライブラリを指定するファイルです。

## srcフォルダ
コードの中身になります。<br>
.cppファイルは実際の関数の中身が書かれています。<br>
.hファイルは関数・定数の宣言や必要な外部ライブラリのインクルードが書かれています。<br>
以下簡単な説明

#### control.cppファイルと.h ファイル
- モータをPWMで駆動させる上でのセットアップ関数と実際に駆動させる関数
- ①madgwick.cppで得たロールピッチヨー(RPY)の姿勢角　②interfaceBLE.cppで受け取った角度あるいは角速度の目標値　を元にPID制御する関数

などが書かれています。<br>
単ループ角度PID制御とカスケード制御の両方の関数があります。

#### debug.cppファイルと.h ファイル
- PCとつないだシリアルモニタでデバッグするための関数

が書かれています。

#### imu.cppファイルと.h ファイル
- 9軸IMUセンサのセットアップ関数
- センサのジャイロ値を校正する関数
- センサの9種の値を読み取る関数
- センサにローパスフィルタを掛ける関数

などが書かれています。<br>
なおDLPFの設定を有効にしたため、代わりに4つ目のローパスフィルタを掛ける関数では実際にはフィルタを掛けていません。

#### interfaceBLE.cppファイルと.h ファイル
- BLE通信する上でのセットアップ関数
- 指令がバイナリ通信で来た際に読み取る関数
- こちらからスマホ側へNOTIFY(送信)するための無線デバッグ関数

などが書かれています。<br>
情報工学や通信工学を専攻しておらず初めての経験だったため、**最も難しく、ほとんどchatGPT頼り**になってしまいました。

#### madgwick.cppファイルと.hファイル
- imu.cpp内の読み取り関数で得た9種(6種)の値を元にRPYの姿勢角を計算する関数

が書かれています。<br>
MadgwickfilterAHRS.hライブラリの関数をそのまま使っています。
- ①9軸(ジャイロ3軸＋加速度3軸＋地磁気3軸)
- ②6軸(ジャイロ3軸＋加速度3軸)
①か②か選択できますが、地磁気はモータ回転で発生する電磁場の影響などで上手く行かなかったため現在は②で計算しています。

#### main.cppファイル
- 上記の各種セットアップに関する関数を実行するvoid setup()関数
- BLE通信で指令値更新 → センサの読み取り → MWFで姿勢角取得 → PID計算してモータ出力 → 各種デバッグ送信　を回すvoid loop()関数

が書かれています。<br>
細かい動作仕様は内のREADMEで閲覧できるYouTube動画も併せてご覧ください。

#### struct.hファイル
今回のドローンコードでは値の受け渡しに構造体を採用しています。<br>
関連するパラメータを一つのまとまりとして管理でき、コードの可読性および拡張性が向上します。<br>  
特にドローン制御では、位置・速度・姿勢といった多次元データを扱うため、構造体によるデータ管理が適していると考えました。 <br> 
<br>
このファイルでは、各種データ構造の定義を行っています。
